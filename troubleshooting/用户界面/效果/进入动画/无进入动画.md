# 无进入动画

## 症状

每次冷启动应用时，整个界面直接「闪现」出来，没有任何淡入、缩放或滑动过渡。

## 原因

`MainActivity.onCreate()` 中没有对根布局做任何动画处理。Android 默认 Activity 启动虽有系统默认过渡（`overridePendingTransition`），但整个 content view 在 `setContentView` 后瞬间可见。

`MainActivity.kt` 之前的代码完全专注于功能初始化（导航栏、insets、主题、ViewModel），忽略了视觉体验——页面内容在布局测量完毕后直接渲染，没有任何 alpha 过渡。

## 修复方案

在 `onCreate` 末尾，对 `android.R.id.content` 根布局做一个淡入动画：

```kotlin
// 首次启动的淡入动画
if (savedInstanceState == null) {
    val contentRoot = findViewById<View>(android.R.id.content)
    contentRoot.alpha = 0f
    contentRoot.post {
        contentRoot.animate()
            .alpha(1f)
            .setDuration(600)
            .setInterpolator(DecelerateInterpolator())
            .start()
    }
}
```

关键点：

- **`savedInstanceState == null`**：确保只在首次创建时播放动画，横竖屏旋转或进程恢复时不重复。
- **`alpha = 0f` 立即设置**：在 post 之前就设，避免布局测量到动画启动的间隙闪现。
- **`post {}`**：等待布局完成后再启动动画，防止测量时根布局还没就位。
- **600ms + DecelerateInterpolator**：平滑淡入，不突兀。
- **`DecelerateInterpolator`** 已有 import，无需新增。

## 验证步骤

1. 杀掉应用进程（确保冷启动）
2. 重新打开应用
3. 观察是否有一道约 0.6 秒的平滑淡入效果
4. 横竖屏旋转后回到应用，不应重复播放动画
5. 按 Home 键再切回，不应重复播放

## 后续注意事项

- 如果未来添加 SplashScreen API（Android 12+），此动画可能需要与 SplashScreen 过渡协调，避免叠加。
- 如要改为从底部滑入（更 Material 的风格），可将 `.alpha(1f)` 替换为 `.translationY(0f)` 配合 `contentRoot.translationY = dp(30).toFloat()`。
- Fragment 之间的切换动画（`FragmentTransaction.setCustomAnimations`）是独立的功能，不影响此 Activity 级进入动画。
