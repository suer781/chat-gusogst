# 无进入动画

## 症状

冷启动应用时，整个界面直接「闪现」出来，没有任何淡入、缩放或滑动过渡感。

## 原因

原实现使用简单的 alpha fade：

```kotlin
contentRoot.alpha = 0f
contentRoot.post {
    contentRoot.animate()
        .alpha(1f)
        .setDuration(600)
        .setInterpolator(DecelerateInterpolator())
        .start()
}
```

问题：
1. **600ms 纯淡入太弱** — 在快速设备上几乎感知不到
2. **无分层交错** — header、内容区、导航栏同时出现，缺乏层次感
3. **`android.R.id.content` 被设置 alpha=0** — 但系统的 windowBackground 仍可见，可能闪现白色/深色背景后才播放淡入

## 修复方案

替换为交错了序贯动画，分四段执行：

```
header 滑入 → 内容缩放淡入 → nav 滑入 → 指示器淡入
  400ms         500ms          400ms        200ms
```

### 关键实现

```kotlin
if (savedInstanceState == null) {
    val header = findViewById<View>(R.id.header)
    val fragmentContainer = findViewById<View>(R.id.fragmentContainer)
    val bNav = findViewById<View>(R.id.bottomNav)
    val navInd = findViewById<View>(R.id.navIndicator)

    // 初始态（用 dimen 资源做偏移，避免布局未测量导致 height=0）
    val offsetY = resources.getDimensionPixelSize(R.dimen.header_height)
    header.translationY = -offsetY.toFloat()
    header.alpha = 0f
    fragmentContainer.alpha = 0f
    fragmentContainer.scaleX = 0.95f
    fragmentContainer.scaleY = 0.95f
    bNav.translationY = resources.getDimensionPixelSize(R.dimen.nav_height).toFloat()
    bNav.alpha = 0f
    navInd.alpha = 0f

    // 序贯触发
    header.post {
        header.animate()
            .translationY(0f).alpha(1f)
            .setDuration(400).setInterpolator(DecelerateInterpolator())
            .withEndAction {
                fragmentContainer.animate()
                    .alpha(1f).scaleX(1f).scaleY(1f)
                    .setDuration(500).setInterpolator(DecelerateInterpolator())
                    .withEndAction {
                        bNav.animate()
                            .translationY(0f).alpha(1f)
                            .setDuration(400).setInterpolator(DecelerateInterpolator())
                            .withEndAction {
                                navInd.animate()
                                    .alpha(1f).setDuration(200).start()
                            }.start()
                    }.start()
            }.start()
    }
}
```

### 关键设计决策

- **不使用 View.height 做偏移** — `onCreate` 中视图尚未测量，直接读取 `height` 返回 0。改用 `R.dimen.header_height` / `R.dimen.nav_height` 资源值
- **`withEndAction` 链式触发** — 每段动画完成后启动下一段，保证顺序执行
- **缩放 0.95 → 1.0** — 比纯淡入更有视觉张力，暗示元素从远处拉近

## 验证步骤

1. 杀掉应用进程（确保冷启动）
2. 重新打开应用
3. 观察三段式入场：header 先滑入 → 内容缩放淡入 → 导航栏滑入
4. 横竖屏旋转后回到应用，不应重复播放动画
5. 按 Home 键再切回，不应重复播放

## 后续注意事项

- 如果未来添加 SplashScreen API（Android 12+），此动画需要与 SplashScreen 过渡协调
- 动画总时长约 1.5 秒，如需更快可缩短各段 duration，或减少 `scale` 量（0.97→1.0）
- 使用 `dimen` 资源而非 `height` 属性是刻意为之——资源值在 XML 中定义，始终可用
