## 这是什么

用户填域名和 API Key，系统自动补全 endpoint，用搭便车测试验证可用性，
按权重公式动态排序端点优先级，故障自动降权切换。

## 用户操作流程

```
1. 进入设置 → API 配置
2. 填入域名（如 api.volcengine.com）和 API Key
3. 系统自动匹配供应商，补全 endpoint
4. 发送第一条消息 → 自动测试端点可用性
5. 后续消息按权重自动选最优端点
```

## 核心能力

| 能力 | 说明 |
|------|------|
| 供应商匹配 | 域名向量搜索，自动补全 endpoint |
| 搭便车测试 | 首条消息即测试，无额外探测请求 |
| 权重排序 | 成功拉满，失败降权，自动切换 |
| 网络诊断 | 重试×3 → ping×3，区分端点/网络问题 |
| 错误累积 | 连续失败越多，惩罚越重 |

## 用户感知

- **正常情况**：无感知，自动选最优端点
- **端点故障**：短暂重试后自动切换，可能有几秒延迟
- **网络故障**：提示网络问题，停止切换，恢复后自动继续

## 供应商字典

- 内置 129 个供应商 / 4774 个模型
- 分类：推荐 / 聚合 / 国产 / 海外
- 向量搜索支持模糊匹配（域名拼错、简写等）
